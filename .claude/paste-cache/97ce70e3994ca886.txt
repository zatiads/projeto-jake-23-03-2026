4. SPRITES — PERSONAGENS VISUAIS
O ai-town usa spritesheets no padrão RPGMaker (48x64px por frame, animação de 3 frames por direção). Você tem 3 opções:

Opção A — Usar sprites do próprio ai-town (mais fácil)
O projeto já vem com personagens prontos. Você apenas mapeia qual sprite corresponde a qual agente no arquivo de configuração. Boa opção para começar rápido.

Opção B — Gerar sprites sci-fi com IA (recomendada)
Acesse: itch.io/game-assets/free/tag-top-down + filtro 'sci-fi'
Ou use Midjourney/DALL-E com prompt: 'top-down RPG character sprite sheet, sci-fi style, 4 directions, pixel art, 48x64px, white background'
Converta para spritesheet com: libresprite.github.io (gratuito)

Opção C — Pixel art manual (mais trabalhoso, mais único)
Editor: Piskel (piskelapp.com) — gratuito, browser
Tamanho: 16x16 ou 32x32 por frame, depois escale
Anime 3 frames: parado / passo esquerdo / passo direito

✔  Recomendação: Comece com Opção A, troque para B depois que tudo estiver rodando.

# Terminal 2 — Frontend Next.js

# Acessa: http://localhost:5173


6. ONDE PERSONALIZAR OS AGENTES NO CÓDIGO
Depois de rodar o projeto pela primeira vez, estes são os arquivos que você vai editar:

ARQUIVO
O QUE EDITAR

Nome, descrição e sprite de cada agente
data/worlds.ts
Layout do mapa, salas, spawn points

Lógica de conversação dos agentes

Como os agentes formam memórias

Modelo LLM usado (troque para claude-3-5-sonnet)

Spritesheets, tileset do mapa, sons


6.1 Exemplo: Como definir um agente no characters.ts
export const characters = [
  {
    name: "NEXUS",
    character: "nexus",   // nome do sprite
    identity: `Você é NEXUS, consultor financeiro especializado em
    tráfego pago. Sempre responde com métricas, ROI e
    sugestões de otimização de campanhas Google/Meta Ads.`,
    plan: "Analisar os dados de campanha mais recentes e compartilhar insights",
  },
  // ... demais agentes
];

6.2 Configurar o modelo Anthropic
Localize onde o modelo é chamado (geralmente em convex/llm.ts ou similar) e substitua pelo modelo Claude desejado:
// Troque o modelo padrão por:
model: "claude-sonnet-4-5"   // bom custo-benefício
// ou
model: "claude-haiku-4-5"    // mais rápido e mais barato
