Na seção Criativos do Jake OS, adicionar suporte ao FLUX.1 Kontext Pro
para edição consistente de imagens existentes.

CONTEXTO DO PROBLEMA:
Quando o usuário sobe uma imagem de referência e pede variações ou
edições (ex: trocar copy, mudar fundo, variar estilo), o modelo atual
(Flux 1.1 Pro) ignora o contexto da imagem e gera algo completamente
diferente. Preciso de consistência real.

SOLUÇÃO: FLUX.1 Kontext Pro via Replicate
Modelo: black-forest-labs/flux-kontext-pro

LÓGICA DE ROTEAMENTO — implementar no backend (app.py):
Se o usuário fez upload de imagem de referência → usar Kontext Pro
Se não tem imagem de referência → usar modelo selecionado normalmente

Chamada Replicate para Kontext Pro:
  replicate.run(
      "black-forest-labs/flux-kontext-pro",
      input={
          "prompt": prompt_instrucao,   # instrução do que mudar
          "input_image": url_imagem_referencia,
          "output_format": "webp",
          "output_quality": 90
      }
  )

DIFERENÇA IMPORTANTE DE PROMPT:
Para Kontext, o prompt não descreve a imagem inteira — ele descreve
só a INSTRUÇÃO de edição. Exemplos:
  ✅ "Troca o texto 'Compre agora' por 'Peça já'"
  ✅ "Muda o fundo para branco, mantém o produto igual"
  ✅ "Adiciona sombra suave embaixo do produto"
  ❌ "Produto de suplemento em fundo branco com iluminação lateral"

Ajustar o engenheiro de prompt (Claude) para detectar se há imagem
de referência e gerar prompts no formato de INSTRUÇÃO ao invés de
DESCRIÇÃO completa. System prompt do engenheiro deve incluir:

"Se houver imagem de referência, gere uma instrução de edição clara
descrevendo APENAS o que deve mudar. Use linguagem direta: 'Troca X
por Y', 'Muda Z para W', 'Remove X mantendo Y intacto'.
Nunca descreva a imagem inteira — só a mudança desejada."

NO FRONTEND (criativos.js):
- Quando há imagem de referência no estado:
  - Label do botão muda para "✏️ Editar Imagem"
  - Placeholder do textarea muda para "O que você quer mudar nessa imagem?"
  - Badge aparece indicando "Modo Edição — Kontext Pro"
- Quando não há imagem:
  - Comportamento normal, "🖼️ Gerar Imagem", seletor de modelo visível

EXEMPLOS DE USO que o sistema deve suportar:
1. Upload de criativo do cliente → "Troca a copy por [novo texto]"
2. Upload de produto → "Coloca num cenário de academia, mantém o produto igual"
3. Upload de imagem gerada → "Muda a cor do fundo para azul marinho"
4. Upload de foto real → "Transforma no estilo de ilustração digital"

Custo estimado: ~$0,04/imagem (mesmo preço do Flux 1.1 Pro atual)
