Prompt de Implementação — Seção "Agenda" no Jake OS
Implemente a seção **Agenda** no Jake OS. Siga rigorosamente o padrão arquitetural e visual do projeto.

---

## 1. CONTEXTO DO PROJETO

**Backend:** Flask monolítico em `app.py`, autenticação por sessão, rotas com prefixo `/api/agenda/`
**Frontend:** SPA com `dashboard.html`, roteamento por hash, JS modular em `static/js/`
**Banco:** Neon (PostgreSQL) via `core/db.py`
**Visual obrigatório:**
- Background: `#050810`, cards: `rgba(10,18,32,.85)` com glassmorphism
- Primária: `#00e5ff` (cyan neon), Verde: `#69f0ae`, Vermelho: `#ff5252`, Dourado: `#ffd740`
- Fontes: Rajdhani (títulos h1-h3), Montserrat (corpo)
- Scrollbar customizada neon, sem dependências externas (apenas emojis/unicode)

---

## 2. O QUE CONSTRUIR

### Visão geral
Uma seção completa de gestão de tempo pessoal com calendário visual, gestão de rotinas fixas e tarefas avulsas. Deve cobrir as visões **Dia**, **Semana** e **Mês**.

---

## 3. BANCO DE DADOS — Neon (PostgreSQL)

Crie as tabelas via `core/db.py` ou migration inline no `app.py` na primeira execução:
```sql
-- Tarefas avulsas e eventos pontuais
CREATE TABLE IF NOT EXISTS agenda_tarefas (
    id SERIAL PRIMARY KEY,
    titulo VARCHAR(200) NOT NULL,
    descricao TEXT,
    categoria VARCHAR(50) NOT NULL DEFAULT 'geral',
    -- categorias: trabalho | pessoal | academia | casa | financeiro | viagem | saude | lazer
    data DATE NOT NULL,
    hora_inicio TIME,
    hora_fim TIME,
    concluida BOOLEAN DEFAULT FALSE,
    prioridade VARCHAR(20) DEFAULT 'media', -- baixa | media | alta | urgente
    criado_em TIMESTAMP DEFAULT NOW()
);

-- Rotinas fixas com recorrência
CREATE TABLE IF NOT EXISTS agenda_rotinas (
    id SERIAL PRIMARY KEY,
    titulo VARCHAR(200) NOT NULL,
    categoria VARCHAR(50) NOT NULL DEFAULT 'geral',
    hora_inicio TIME NOT NULL,
    hora_fim TIME,
    dias_semana VARCHAR(20) NOT NULL,
    -- formato: "1,3,5" (seg=1, ter=2, qua=3, qui=4, sex=5, sab=6, dom=7)
    ativa BOOLEAN DEFAULT TRUE,
    cor VARCHAR(7) DEFAULT '#00e5ff',
    criado_em TIMESTAMP DEFAULT NOW()
);
```

## 4. BACKEND — `app.py`

Adicione as seguintes rotas Flask (todas protegidas por `@login_required`):
```python
# TAREFAS
GET  /api/agenda/tarefas?data=YYYY-MM-DD          → tarefas do dia
GET  /api/agenda/tarefas?mes=YYYY-MM              → tarefas do mês inteiro
POST /api/agenda/tarefas                           → criar tarefa
PUT  /api/agenda/tarefas/<id>                      → editar tarefa
DELETE /api/agenda/tarefas/<id>                    → deletar tarefa
PATCH /api/agenda/tarefas/<id>/concluir            → toggle concluída

# ROTINAS
GET  /api/agenda/rotinas                           → listar todas rotinas ativas
POST /api/agenda/rotinas                           → criar rotina
PUT  /api/agenda/rotinas/<id>                      → editar rotina
DELETE /api/agenda/rotinas/<id>                    → deletar/desativar rotina

# AGENDA COMBINADA — retorna rotinas + tarefas de uma semana
GET  /api/agenda/semana?inicio=YYYY-MM-DD          → dados da semana (7 dias a partir da data)
```

Todos os endpoints retornam JSON `{"success": true, "data": [...]}` ou `{"success": false, "error": "..."}`.

### Funções obrigatórias
- `initAgenda()` — inicializa a seção, carrega dados
- `renderCalendarioMes(ano, mes)` — grid 7×6 com dias do mês
- `renderSemana(dataInicio)` — grid 7 colunas com blocos de horário (06h–23h)
- `renderDia(data)` — timeline vertical do dia com horários
- `abrirModalTarefa(data, hora)` — modal para criar/editar tarefa
- `abrirModalRotina()` — modal para criar/editar rotina
- `toggleConcluida(id)` — marca tarefa como feita (sem reload)
- `navegarPeriodo(direcao)` — anterior/próximo (−1 ou +1)
- `getCorCategoria(categoria)` — retorna cor hex por categoria

## 9. COMPORTAMENTOS DETALHADOS

### Visão Mês
- Grid 7 colunas × 6 linhas com dias do mês atual + dias do mês anterior/seguinte (acinzentados)
- Dia de hoje destacado com borda cyan + glow
- Cada célula mostra até **3 eventos** como pills coloridas (cor da categoria)
- Se houver mais de 3: mostrar "+N mais" clicável
- Clicar num dia abre o **Day Panel** abaixo do calendário com a lista de tarefas daquele dia
- Clicar num dia vazio abre o modal de nova tarefa com a data pré-preenchida

### Visão Semana
- 7 colunas (seg–dom) com header mostrando dia e data
- Timeline de **06h às 23h** (17 horas) com slots de 60px cada
- Rotinas aparecem automaticamente nos dias corretos com opacidade reduzida e borda esquerda para diferenciá-las
